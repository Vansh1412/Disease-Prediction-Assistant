Docs/Overview

MediSense AI

Production-quality disease prediction platform — documentation for v1.0

ℹ️

MediSense AI is a Streamlit-based medical diagnosis assistant combining three scikit-learn classifiers, a local Ollama LLM (Qwen 2.5:3B), and an explainable AI report system into one cohesive application.

What Is MediSense AI?

A patient describes symptoms in natural language. The app processes them through an intelligent chatbot, runs three ML models, and delivers a hospital-grade diagnostic report with XAI reasoning, specialist referrals, and lifestyle recommendations.

💬
NLU Consultation
Qwen 2.5:3B via Ollama extracts structured symptoms from natural conversation.
🧠
3-Model Ensemble
Logistic Regression, Decision Tree, and KNN trained on 247K+ records.
📄
XAI Reports
AI-generated reasoning explaining each prediction in clinical language.
📊
Analytics
Real-time Plotly charts tracking predictions, symptoms, and model performance.
🔒
Authentication
SQLite-backed user accounts with SHA-256 password hashing.
⚡
Emergency Detection
Rule-based escalation to specialist before ML inference completes.

Docs/Quickstart

Quickstart

Get MediSense AI running in under 5 minutes.

Prerequisites

  • Python 3.11+
  • Ollama installed and running: ollama serve
  • Qwen model pulled: ollama pull qwen2.5:3b
  • Pre-trained models in models/general/

Installation

cd Disease-Prediction-Assistant
python -m venv venv
venv\Scripts\activate         # Windows
source venv/bin/activate      # macOS/Linux
pip install -r requirements.txt
streamlit run app/main.py

Developer Mode

http://localhost:8501/?dev=true
⚠️

Ollama must be running on port 11434 for consultation and XAI reasoning features. ML prediction works without Ollama.

Docs/Report

Report Generation

The Report page reads st.session_state.prediction_results and assembles a complete HTML document rendered via st.markdown(html.strip(), unsafe_allow_html=True).

ℹ️

The HTML is pre-built and .strip()'d before passing to st.markdown. Required because CommonMark treats 4+ leading spaces as code blocks.

def load_models():
    models = {

        "Decision Tree":      joblib.load(MODEL_DIR / "DecisionTree.pkl"),

    return models, encoder

# src/chatbot/llm_engine.py
import ollama
def ask_llm(messages, stream=True):
    response = ollama.chat(model="qwen2.5:3b", messages=messages, stream=stream)
    if stream:
        yield from response
    else:
        return response["message"]["content"]

def get_connection() -> sqlite3.Connection:
    conn = sqlite3.connect(DB_PATH, check_same_thread=False, timeout=10)
    conn.execute("PRAGMA journal_mode=WAL")    # Crash-safe write-ahead log
    conn.execute("PRAGMA busy_timeout=5000")   # Wait up to 5s on lock
    conn.execute("PRAGMA foreign_keys=ON")     # Enforce FK constraints
    return conn

Docs/Infrastructure/Caching

Caching Strategy

FunctionDecoratorWhy
load_models()@st.cache_resourceSklearn objects must persist without serialisation
load_feature_names()@st.cache_dataCSV read is expensive; columns are immutable
load_feature_index()@st.cache_dataDerived from feature names; also immutable
_read_css()@st.cache_data54KB CSS file read once per process
init_db()@lru_cacheTable creation only needed once per process
ℹ️

cache_resource shares objects across ALL sessions. cache_data serialises and gives each caller their own copy. Use cache_resource for models; cache_data for dataframes and primitives.

Docs/Infrastructure/Security

Security

ControlImplementation
Password storageSHA-256 hash — no plaintext stored
SQL injectionAll queries use parameterised placeholders (?)
HTML injectionescape_html() used for all user data in HTML contexts
Authenticated routingmain.py gates on st.session_state.logged_in
Data localityAll data stays on local machine
🚨

No salt on password hashes. For real patient data, replace SHA-256 with bcrypt or argon2 with per-user salts.

⚠️

No RBAC on developer page. Any user with ?dev=true can access system diagnostics. Mitigate with role-based access control.

Docs/Deployment

Deployment

⚠️

MediSense AI requires Ollama locally. Not suitable for Streamlit Community Cloud without replacing the LLM backend.

streamlit run app/main.py \
    --server.port 8501 \
    --server.headless true \
    --server.enableXsrfProtection true

Docker

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8501
CMD ["streamlit", "run", "app/main.py", "--server.headless=true"]
